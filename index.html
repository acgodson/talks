Stateless vs Stateful

The most important design decision for scalable systems.

IMAGINE
Stateless = vending machine. Every button press works the same, no history remembered. Stateful = a human cashier who remembers your running tab.
📭 Stateless
Each request carries everything the server needs. Any server can handle any request.
✓ Scales horizontally
📬 Stateful
Server holds session in memory. Fast, but tied to one server — breaks if that server dies.
⚠ Hard to scale
The fix: shared session store

Stateless auth

How JWT makes services stateless

Server B Receives session_id=abc123. Never seen it — session lives in Server A's memory. User logged out. ✗
vs. with JWT →
Server Creates token: user_id: u_42, role: rider, exp: +15min. Signs it. Sends it back. Stores nothing.
Server B Receives token. Verifies signature. Reads user_id + role directly. No DB lookup. ✓
Server C Same result. Any server in the fleet handles any JWT. Fully stateless. ✓
Header
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
Decodes to: { "alg": "HS256", "typ": "JWT" }
Payload — your data
eyJ1c2VyX2lkIjoidV80MiIsInJvbGUiOiJyaWRlciJ9
Decodes to: { "user_id": "u_42", "role": "rider", "exp": 1712000000 }
Signature — tamper proof
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c
HMAC(header + payload, secret_key). Edit payload → signature breaks → rejected.
📖 Payload is readable
Anyone can base64-decode it. Never put passwords in JWT.
🔒 Payload can't be changed
Change role:rider → role:admin? Signature fails instantly.

08 / 18 Auth & Authorization Instructor D
Prevention

Preventing Abuse & Data Leaks

🤖
Bot & DDoS attacks
Rate limiting + IP blocking + CAPTCHA. Bolt/Uber use ML anomaly detection for surge abuse.
🗃
API scraping
Pagination limits, auth on all endpoints, response field filtering. Don't expose internal IDs.
🔓
Broken object-level auth (BOLA)
The most common API bug. User A shouldn't access user B's trip data. Always check ownership.
📤
Sensitive data exposure
Encrypt in transit (HTTPS) + at rest. Never log tokens or passwords. Mask PII in responses.
📍
Location data leaks
Ride-hailing specific. Fuzzy driver positions before pickup confirmed. Purge raw GPS logs after trip.

Surge Pricing — What triggers it?

When demand (riders) outpaces supply (drivers), the system applies surge. Watch the load balancer distribute driver-match requests.

Lagos Island — request zone
Load balancer role here
